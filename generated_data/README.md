## File Formats

- **`.p` files**: Python pickle format (binary) containing dictionaries and numpy arrays
- **`.npz` files**: NumPy compressed archive format containing multiple arrays

## Regenerating Data Files

If any of these files are missing or need to be regenerated:

1. Ensure external data sources are available (see main README)
2. Run processing scripts in the order specified in `processing/README.md`
3. Estimated time: 48-72 hours with parallelization on HPC system

## Data Structure

Each `.p` file contains a pickle-dumped Python dictionary with keys like:

- `dsd_params` - Gamma distribution parameters
- `microphysical_vars` - Cloud microphysics (q_c, q_r, q_i, etc.)
- `indices` - Spatial/temporal indices
- `timestamps` - Time information
- `reflectivity` - Radar reflectivity values
- etc.

Details are documented in individual processing script notebooks.

## Deletion Policy

Safe to delete if:
- You have backup copies
- You plan to regenerate from source data
- You have the external data sources to re-process

NOT safe to delete if:
- You need to run analysis scripts (analysis scripts depend on these files)
- Processing scripts are not available or data sources are inaccessible

## Integration with Analysis Scripts

Analysis scripts automatically load `.p` files:

```python
import pickle

with open('generated_data/wrf_dsd_props_cu_only.p', 'rb') as f:
    data = pickle.load(f)
```
